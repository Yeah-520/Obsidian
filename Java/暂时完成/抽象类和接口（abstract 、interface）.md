# 抽象类(abstract)
>包含`abstract`抽象方法的类
>无法实例化~~`new`~~
>**抽象类不一定有抽象方法**
>抽象方法不能有方法主体 `{}`
```java
	abstract class Person {
		public abstract void run();
	}
```
## 要求
- 继承抽象类的==子类==**必须重写父类所有的抽象方法**，否则，该子类也必须声明为抽象类
## 作用
- 定义规范，制定规则
- 约束子类必须执行的行为方法
- 避免遗忘行为方法
- 具体的**实现方法**逻辑由不同的**子类实现**
# 接口(interface)
>**方法契约** + **常量**容器
>**没有***方法实现*或成员变量（JDK8以后：新增默认方法（**default method**））
```java
public class Main {  
    public static void main(String[] args) {  
        Car car = new Car();  
        car.run();  // 当前时速：100
    }  
}  
// 使用关键字 interface 定义接口
interface Runnable {  
    void run();  
    int MAX_SPEED = 100;  
}  
// 使用关键字 implements 来调用接口
class Car implements Runnable {  
  
    @Override  
    public void run() {  // 必须重写（实现）接口中的方法
        System.out.println("当前时速：" + MAX_SPEED);  
    }  
}
```
## 实现方法
- 使用 `implements`关键字
```java
class 类名 implements 接口1,接口2,接口3...{
}
```
### 要求
- 接口体现的是一种规范，接口对实现类是一种强**制性的约束**，需要**强制重写**或定义为抽象类
## 作用
- 实现多继承
- 接口就是一个**规则**，**独立**于继承体系外的规则
- 接口是 **规范方法命名**(规范行为)
- 类需要实现接口里的**所有**行为方法
- 一个类可以**实现多个接口**
## 新增
- JDK8以后：新增默认方法（**default method**）
- 解决接口升级的问题
- 静态方法只能通过接口名调用，不能通过实现类名或者对象名调用
```java
// public default 返回值类型 方法名(参数列表) {方法体}
public default void show(){
    System.out.println("新增默认方法 default method");
}
```
## 抽象类和接口的区别

|          |       抽象类       |        接口        |
| :------: | :-------------: | :--------------: |
|   方法实现   | 可以有实现的方法和非抽象方法  |   只有方法签名，无方法实现   |
| **构造函数** |   可以**有**构造函数   |   **无**法定义构造函数   |
| **成员变量** |   可以**有**成员变量   | 只能定义常量，**无**成员变量 |
| **继承**关系 | 子类只能**继承一个**抽象类 |  类可以**实现多个接**口   |
|   功能实现   |    提供对类的部分实现    |    定义契约和行为规范     |

|      关系类型      |  涉及主体   |     关键字      |                  核心规则                  |
| :------------: | :-----: | :----------: | :------------------------------------: |
| 实现（Implements） | 类 ↔ 接口  | `implements` | 一个类可以**实现多个接口**，必须重写接口中所有的抽象方法（除非是抽象类） |
|  继承（Extends）   |  类 ↔ 类  |  `extends`   | 一个类只能**继承一个父类**（单继承），可继承父类的非私有属性 / 方法  |
|  继承（Extends）   | 接口 ↔ 接口 |  `extends`   |      一个接口可以**继承多个接口**，整合多个接口的抽象方法      |
